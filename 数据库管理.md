# 例程管理：

##  一、Oracle 启动

### 一步启动（最常用）

```
STARTUP;
```

等价于：

- NOMOUNT
- MOUNT
- OPEN（自动完成）

------

### 分阶段启动

------

####  ① NOMOUNT（实例启动）

```sql
STARTUP NOMOUNT;
```

或：

```sql
ALTER DATABASE NOMOUNT;
```

------

####  ② MOUNT（挂载数据库）

```
ALTER DATABASE MOUNT;
```

------

####  ③ OPEN（打开数据库）

```
ALTER DATABASE OPEN;
```


------

###  三阶段关系图

```
STARTUP NOMOUNT
      ↓
读取参数文件（pfile/spfile）
      ↓
MOUNT
      ↓
读取控制文件
      ↓
OPEN
      ↓
数据库可用
```

------

### 常见补充命令

------

#### 查看数据库状态

```
SELECT status FROM v$instance;
```

------


## 二、Oracle 关闭

------

### 正常关闭（NORMAL）

```sql
SHUTDOWN NORMAL;
```

特点：

- 不允许新连接
- 等所有用户退出
- 完成当前操作才关闭

------

### 事务性关闭（TRANSACTIONAL）

```sql
SHUTDOWN TRANSACTIONAL;
```

特点：

- 等当前事务提交/回滚后关闭
- 不等待空闲连接

------

### 立即关闭（IMMEDIATE）

```
SHUTDOWN IMMEDIATE;
```

特点：

- 立即断开所有连接
- **未提交事务自动回滚**
- 数据文件正常关闭

------

###  强制关闭（ABORT）

```sql
SHUTDOWN ABORT;
```

特点：

- 直接终止数据库进程
- 不做任何清理
- 下次启动需要实例恢复

------



## 三、MySQL 关闭

MySQL 没有 Oracle 那么细的 shutdown 分级，主要是：

------

###  正常关闭

```sql
mysqladmin -u root -p shutdown
--- 或：
systemctl stop mysqld
```

特点：

- 等当前操作完成
- 安全关闭

------

###  立即关闭

```sql
systemctl stop mysqld
--- 或：
mysqladmin shutdown
```

特点：

- 关闭服务并回滚事务

------

### 强制终止（Abort 类似）

```sh
kill -9 mysqld_pid
```

### 特点：

- 直接杀进程
- 可能导致 InnoDB crash recovery


------



## 四、 Oracle的只读数据库 & 受限数据库

| 类型                             | 功能                    | 启动/开启方式                                                | 关闭方式                                   | ALTER语句                                         | 是否影响用户                      | 说明             |
| -------------------------------- | ----------------------- | ------------------------------------------------------------ | ------------------------------------------ | ------------------------------------------------- | --------------------------------- | ---------------- |
| 只读数据库（READ ONLY）          | 数据库只能查询不能修改  | `STARTUP MOUNT;` + `ALTER DATABASE OPEN READ ONLY;`          | `SHUTDOWN` 后重启 NORMAL                   | `ALTER DATABASE OPEN READ ONLY;`                  | ❌ 不能DML（INSERT/UPDATE/DELETE） | 常用于备份、报表 |
| 读写数据库（READ WRITE）         | 默认正常模式            | `STARTUP;`                                                   | `SHUTDOWN`                                 | `ALTER DATABASE OPEN READ WRITE;`                 | ✅ 完全可用                        | 默认模式         |
| 受限数据库（RESTRICTED SESSION） | 仅管理员/授权用户可访问 | `STARTUP RESTRICT;` 或 `ALTER SYSTEM ENABLE RESTRICTED SESSION;` | `ALTER SYSTEM DISABLE RESTRICTED SESSION;` | `ALTER SYSTEM ENABLE/DISABLE RESTRICTED SESSION;` | ⚠️ 普通用户不可连接                | 用于维护/升级    |
| MOUNT只挂载状态                  | 数据库未打开            | `STARTUP MOUNT;`                                             | `ALTER DATABASE OPEN;`                     | 无                                                | ❌ 无用户访问                      | 控制文件已读     |
| NOMOUNT实例状态                  | 仅启动实例              | `STARTUP NOMOUNT;`                                           | `SHUTDOWN`                                 | 无                                                | ❌                                 | 初始化阶段       |

------



###  1、只读数据库

------

#### 启动到只读模式（标准写法）

```
STARTUP MOUNT;
ALTER DATABASE OPEN READ ONLY;
```

------

#### 修改为只读（运行中切换）

```
ALTER DATABASE OPEN READ ONLY;
```

- 必须先处于 MOUNT 或关闭状态
- 不能在已有DML事务时切换

------

####  恢复为读写模式

```
ALTER DATABASE OPEN READ WRITE;
```

------

#####  

### 2、数据库受限模式

------

#### 启动受限数据库

``` sql
STARTUP RESTRICT;
```

``` sql
STARTUP;
ALTER SYSTEM ENABLE RESTRICTED SESSION;
```

------

#### 开启/关闭受限模式（运行中）

``` sql
--- 开启
ALTER SYSTEM ENABLE RESTRICTED SESSION;

--- 关闭
ALTER SYSTEM DISABLE RESTRICTED SESSION;
```

------

#### 受限模式特点

| 项目     | 说明                   |
| -------- | ---------------------- |
| 普通用户 | 不能连接               |
| DBA用户  | 可以连接               |
| 用途     | 维护、升级、数据修复   |
| 数据读写 | 正常（只限制连接权限） |
