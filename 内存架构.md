# 管理表空间和数据文件

1      数据库存储的层次结构

数据库

表空间      /      数据文件

段        ----存储结构  (dba\_segments)

区        ----oracle最小的空间分配单位

块        ----oracle最小的I/O单位



主块   ----创建数据库时定义

从块   ----创建表空间时定义



2        区管理

1）    数据字典管理

2）    本地管理

```
select TABLESPACE_NAME,EXTENT_MANAGEMENT from dba_tablespaces;

create tablespace dict_manage datafile '/u01/app/oracle/oradata/sales/dict01.dbf' size 10m EXTENT MANAGEMENT DICTIONARY;
```



如果system表空间是本地管理的，那只能创建本地管理的表空间，如果system表空间是数据字典管理的，既可以创建本地管理的表空间，也可以创建数据字典管理的表空间。

本地管理的表空间的区大小：

uniform    ----用户定义区大小（每个区一样大小）

autoallocate   ----系统定义区大小（区可能一样大小，也可能不一样大小）

8     临时表空间组

查看临时表空间组

```
select * from dba_tablespace_groups;
```

将临时表空间添加到组

alter tablespace TEMP1 tablespace group group1;

\--添加temp2,temp3到组group1

```sql
SQL> create temporary tablespace temp2 tempfile '/u01/app/oracle/oradata/sales/temp201.dbf' size 10m tablespace group group1;

SQL> create temporary tablespace temp3 tempfile '/u01/app/oracle/oradata/sales/temp301.dbf' size 10m tablespace group group1;

--更改属组

SQL> alter tablespace temp1 tablespace group group2;

--删除组员（当删除的组员是最后的一个时，会默认删除对应的组）

SQL> drop tablespace temp1;

--使临时表空间退出临时表空间组

SQL> alter tablespace temp3 tablespace group '';

--切换default tablespace

SQL> alter database default temporary tablespace group1;


```

9    表空间状态

```sql
select tablespace_name,status from dba_tablespaces;
```

1）  联机可读写

2）  只读  --alter tablespace demo read only;



3\)    脱机



不允许脱机的表空间：

1）system表空间

2）当前的撤销表空间

3）临时表空间



脱机模式：

1） 正常脱机（默认）  offline/offline normal    ----联机时不需要介质恢复（要求该表空间内所有的数据文件联机）



2\)   临时脱机          offline temporary                ----联机时可能需要介质恢复，也可能不需要介质恢复（该表空间内的数据文件可以联机，也可以脱机）

3\)   立即脱机          offline immediate                ----联机时需要介质恢复

1） 修改原数据文件大小

（1） 自动扩展

```

alter database datafile '/u01/app/oracle/oradata/sales/users101.dbf' autoextend on next 10m maxsize 50m;

```

（2） 手工扩展

```

```

2） 增加新的数据文件

```

select file_name from dba_data_files where tablespace_name='USERS1';


```



13   移动或重命名数据文件

1）  12c之前

（1） 保证数据文件不在被使用

（2） 用操作系统的命令移动或重命名数据文件

（3） 更新控制文件



2）  12c起

在线移动或重命名

```sql
alter database move datafile '/u01/app/oracle/oradata/sales/users102.dbf' to '/u01/demo/users02.dbf';        ---移动，不保留原数据文件

alter database move datafile '/u01/demo/users02.dbf' to '/u01/app/oracle/oradata/sales/users102.dbf' keep;    ---复制，保留原数据文件
```



# 作业：数据文件在没有备份的情况下恢复

前提

1）   不能是system数据文件

2）   数据文件产生后，控制文件未重建过

3）   日志要齐全


## 1. 创建表空间和表

```sql
SYS@sales>select name from v$tablespace;
NAME
------------------------------
SYSTEM
SYSAUX
UNDOTBS1
USERS
TEMP

SYS@sales>create tablespace demo datafile '/u01/app/oracle/oradata/sales/demo01.dbf' size 10m;

SYS@sales>select name from v$tablespace;
NAME
------------------------------
SYSTEM
SYSAUX
UNDOTBS1
USERS
TEMP
DEMO

SYS@sales>create table scott.test2(id int) tablespace demo;
SYS@sales>insert into scott.test2 values(100);
SYS@sales>insert into scott.test2 values(200);
SYS@sales>commit;
SYS@sales>alter system checkpoint;

SYS@sales>select * from scott.test2;
        ID
----------
       100
       200

```
